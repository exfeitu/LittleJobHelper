# AI 协作提示词模板

下面这些提示词可以直接复制给 AI，用来持续开发这个项目。

---

## 1. 新功能开发模板

```text
你现在是这个项目的资深全栈工程师。
请先阅读相关文件，再实现这个功能，不要凭空猜测。

目标功能：
[在这里写明确需求]

要求：
1. 先说明你准备查看哪些文件。
2. 阅读后再开始改代码。
3. 只做和当前需求直接相关的修改，不要额外重构。
4. 改完后说明：
   - 修改了哪些文件
   - 为什么这样实现
   - 还有哪些后续工作没做
5. 如果新增字段，请同步考虑搜索、筛选、导出。
```

---

## 2. Supabase 接入模板

```text
请为当前项目接入 Supabase。
先阅读现有的类型、页面和示例数据文件，再进行修改。

本次只完成以下内容：
1. 环境变量模板
2. Supabase 客户端初始化
3. 登录页基础版本
4. events 和 todos 的读取

要求：
- 不要一次性做太多功能
- 保持 TypeScript 类型完整
- 如果需要新增目录或文件，保持结构清晰
- 改完后告诉我数据库表结构应该如何建立
```

---

## 3. 待办功能增强模板

```text
请增强待办模块。
先阅读待办相关组件、类型和工具函数。

目标：
- 支持新增待办
- 支持编辑 department、contactPerson、remarks
- 支持新增子任务
- 支持父任务状态自动计算
- 支持按部门和联系人搜索

要求：
- 不要破坏现有页面结构
- 优先复用现有类型和逻辑
- 如果修改数据结构，请同步更新导出逻辑
```

---

## 4. App 迁移模板

```text
请基于当前 Web 项目的领域模型，为我规划并创建 React Native + Expo 移动端基础代码。

目标：
- 登录页
- 今日待办页
- 待办列表页
- 日程记录页

要求：
1. 尽量复用现有 TypeScript 类型定义
2. 先给出目录结构和技术方案，再写代码
3. 保持后续可接入 Supabase
4. 不要实现与当前目标无关的高级功能
```

---

## 5. Bug 修复模板

```text
请帮我排查并修复这个项目中的问题。
先阅读相关文件，再定位原因。

问题现象：
[在这里粘贴报错或异常表现]

要求：
1. 先说明问题最可能出在哪些文件
2. 阅读后再给出修复
3. 修复保持最小改动
4. 修复后告诉我根因是什么
```
